Сущности и их аттрибуты

  • Student - пользователь-ученик. В таблице all_members имеет значение поля status равное "student". При инициализации сущности Student преподавателем ей присваивается собственный member_id. При первичной авторизации считывается tg_id и записывается в таблицу all_members.
  • Teacher - пользователь-преподаватель. В таблице all_members имеет значение поля status равное "student". При первичной авторизации (регистрации) ему присваивается собственный member_id.
  • Note - файл-конспект.

Действия с БД (db module)

get-actions:

  • member_id - возвращает member_id по его tg_id. Если такого пользователя нет, то вызывает исключение. Единственная функция в get-actions, которая обрабатывает tg_id.
  • status_by_id - возвращает статус пользователя ("student"/"teacher") по его member_id. Если такого пользователя нет, то вызывает исключение.
  • name_by_id - возвращает имя пользователя по его member_id
